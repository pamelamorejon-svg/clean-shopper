06

Usage Rules

  1. Never hardcode a hex value. Use the token name from tailwind.config.js.
  2. Never hardcode a pixel font size. Use the type scale token classes.
  3. Never hardcode spacing. Use the spacing scale token classes.
  4. Accent (forest green) is for the single most important interactive element per view. Not used for backgrounds or decoration.
  5. Neutral-900 for primary text. Neutral-600 for secondary text. Never use pure black (#000000).
  6. Semantic colors carry meaning. Success = clean/safe only. Error = avoid/unsafe only.
  7. Surfaces must use neutral-50 or secondary-cream. No bright white (#FFFFFF) backgrounds.
  8. Cormorant Garamond for display, h1–h3 (editorial). Jost for h4, body, small, micro (interface).
  9. No saturated colors, loud gradients, or decorative patterns. Tone-on-tone only if texture is needed.
  10. Spacing should feel generous. When in doubt, add more vertical space.
